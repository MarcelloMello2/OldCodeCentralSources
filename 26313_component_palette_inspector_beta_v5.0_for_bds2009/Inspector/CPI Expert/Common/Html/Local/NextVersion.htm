New Features in CPIV50

INI files

Most if not all anti-viruses software packages limit applications from writing to registry to protect computers from viruses which write to system registry. To avoid this limitation, CPI V50 uses ini file for storing its status instead of registry in "PackageIniFile" as shown in figure below for BDS2007 and limits registry for none changing keys.

CPI V50 only keeps a few keys in registry that are set during installation.  So registry writing can be enabled during installation and then late disabled.  Figure below shows registry keys for CPI in BDS2007 that keeps where CPI is installed in system storage.

Quick Dynamic Navigation in objects inspector

Now CPI automatically sets quick navigation tools based on selected component.  Each component has its own navigation tools which depends on component prosperities and user preferences.  CPI allows users to setup navigation tools for all components and to save them permanently.  If no setup is found, CPI will use its default setup.

CPI now offers 11 navigation tools. Two of these 11 tools are static tools and the remaining ones are dynamic tools. Dynamic tools changes depending on selected component and static tools do not change based on selected tool in component designer.  The two static tool are, "home" tool and arbitrary tool .  The the remaining nine tools are dynamically set by CPI. Figure on right shows nine dynamic tools on top in. The two static tools are located on sides of second tools bar.

The home tool always navigate to "Name" property.  The arbitrary tool has a multi usage. It can be set by user and does not change with component selections on form editor.

CPI added a new page for setting up dynamic navigations tools. with 'NAV"  (NAVigation) caption as shown in figure below.  The "search" page now is renamed to be "Find" page to allocate space for NAV page.
 
The "NAV" page is shown in right side.  The page uses its own tools bar and a list box.  The box lists the properties of selected components in form editor.  The box shows a bitmap for each property and a counting index and a check box.  Users can check only nine properties and then click save tool which saves the dynamic navigation tools of selected component.  Each time the component is selected on form editor, CPI shows these on its top tool bar.  When a user click any of these tools, CPI will navigate to its property in object inspector.

The page provides a counter tool that counts checked properties.  When the count exceeds "9" CPI informs user and starts to uncheck extra tools from top list.

Prosperities with no bitmaps assigned uses  this bitmap.  Events properties are displayed with blue color and none-event prosperities are displayed in black color.

This dynamic approach gives users a maximum flexibility and a quick navigation means in object inspector.

CPI OP page in ToolForm

Delphi IDE offers "context-menu" in form editor to "Edit" and "Navigate" in component structure.  In the menu, the IDE offers items for editing like "New Button" (e.x. TToolbar) and offers items for navigation like "Next Page" (ex. TPageControl).  With more components installed and more third party intervention in the menu, the menu becomes crowded and makes the overall navigations of users in IDE slow.

To enhance this process, CPI adds a NEW page called "OP" (OPerations) page in ToolForm.  This page has two sections.  dynamic section and static section. The dynamic section provides users with "operations" that are specific to selected component.  These operations include "Editing" and "Navigation".  In static section, the page provides "operations" that are "common" on all components and are not specific to any component.
 
CPI "OP" page is shown in right figure with its two sections.   In dynamic section, it shows the selected component class name "TToolButton". and list of operations.  The list can be scrolled to locate desired operation.  More editing operations and navigations operations can be added than present in picture. 
Clicking any operations activates that operations. 

For example, clicking "Select page from menu"  for a "TPageControl" components generates a popup menu with all pages from which user can select randomly (Random-Access-Selection).  Figure below shows popup menus of main page control in old versions of CPI  main form.  As you see selecting any page is quick and random.

Image List Editor (ILE)

Working with image lists is a task that no programmer can a void.  Tool bars, menus, trees ...etc all use image lists.  Due to my needs in programming I developed the ILEE to handle image lists because I use them heavily in my programs.  The ILEE enable users to do the followings:

  1. Drag and drop images between lists

  2. Automatically creates image lists in form editor from selected images in a list

  3. filter repeated bitmaps if found in a list

  4. move images from one list to another list

  5. copy images from one list to another list

  6. identify similar images in two lists (bitmap-comparison)

  7. identify none similar images in two lists

  8. delete selected images in a list

  9. send deleted images to a recycle list

  10. provides basic image flipping routines

  11. replace images

  12. modify image by an external image editor (MS-Paint is used as default editor)

  13. Save list to a single bitmap file that can be loaded later by Delphi image list editor

  14. Save into special binary files

  15. Save selected images to a single bitmap file that can be loaded later by Delphi image list editor

  16. Export selected images to a subdirectory in separate files

  17. more ...
