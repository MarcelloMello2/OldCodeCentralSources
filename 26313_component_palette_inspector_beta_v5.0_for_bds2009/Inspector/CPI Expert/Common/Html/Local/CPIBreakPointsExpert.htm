CPI Breakpoints Expert

CPI BPE identifies breakpoints by unique IDs and append a BP comment at the end of line code of each BP.  The comment has this form: //{$BP D2J9F9F4I64C3J9H1J7B1;tag;Hint} as shown in figure below
CPI adds only three fields in $BP comment which are ID;tag;hint
 
List-view

List view lists loaded PBs with all defined fields by BDS and CPI.

Fields order can be customized from setup page.  Each column in view has its own color. From setup page, you can customize these colors.  Colors are used to simplify reading data in columns.
Tree-view

Tree view lists loaded PBs with all defined fields by BDS and CPI in a tree structure.

BPs are grouped in different levels.  The first level is their files and second level in their groups.  BPs with no groups are listed in "Unclassified" node

BP properties are grouped in three nodes: 

  1. Text Node
    Contains all string properties

  2. Numerical Node
    Contains all numerical properties

  3. Actions Node
    Contains all Boolean properties

Some important properties are displayed with same level of BP as shown on right figure.
Text-view

Text view lists loaded PBs the way they are stored in project *.dsk file.

The view provides an editor to view full selected BP.

 
Stack-view

Stack view lists loaded PBs with all fields.  Fields of a BP are viewed when expanding wanted BP.  A BP with a '+' indicates a collapsed BP and a BP with a '-' indicates an expanded BP.

Multiple expansions is possible. BPs  fields or properties can be edited in this view.

 
   
Synchronize-view

Synchronize view is used to compare imported or loaded BPs to the ones already exist in active project group. The view has two sections left and right.  Left view shows loaded BPs and right view shows installed BPs.

Selecting a BP in any view, result in selecting the same BP in the other view if it exists.

Color Legends:
Red Uninstalled BP
Dark green  Installed and enabled BP
Light green  Installed and disabled BP

 
Edit-view

Edit view is used for editing BPs and viewing search results.  All search results are appended to this view.

Loaded BPs can be loaded in this view directly.

 
Search-view

Search view is used for search and replace operations.  The search has two types.  Simple search and complex search.

In simple search, BPE searches for a keyword in selected fields for search.  In complex search, BPE searches for different keywords in different fields.

In replace operation, BPE can replaces different fields for all BPs in search view.
   
Setup-view

Setup view is used to customize some of BPE operations.  View has four sheets: First, is "General" sheet which is used to set fields required for listing.   Second, "flags" sheet is used to control flags.  Third, "colors" sheet used to set colors of BPE views.  Forth, is "Fields" sheet, which is used to set needed fields to included in BP-folding block.

 
   

CPI BPE enables users to do the followings

  1. Breakpoint Loading
    1. Direct BP loading from active project by OTAPI
       
  2. Breakpoints Exporting
    1. Exports breakpoints to INI files
    2. Export breakpoints to BIN files
    3. Export breakpoints to TXT files
    4. Export breakpoints to DSK  file
       
  3. Breakpoints Importing
    1. Import breakpoints from INI files
    2. breakpoints from BIN files
    3. breakpoints from TXT files
    4. breakpoints from DSK  file
       
  4. Breakpoints Installing
    1. Installs imported break points to active project by two methods
      1. file-Line Method depends on line position
      2. file-ID Method independent of line position
         
  5. Breakpoint Searching
    1. Simple Search for BPs
    2. Complex search for BPs
       

  6. Breakpoint Replacing

    1. Replace any field insid BPs
       

  7.  Breakpoint Scanning

    1. Scans source code for BPs
    2. Scans source code for BPs collectively
       

  8. Breakpoint Re-Categorizing

    1. Rearrange BP groups
    2. Send selected BPs to a group
    3. Send selected BPs to a Enable group
    4. Send selected BPs to a Disable group
       
  9. Breakpoint General
    1. Loading source lines
    2. Appending/Loading/Removing break points hints
    3. Appending/Loading breakpoints IDs
    4. Removing hints from source lines
