CPI Help Expert
HE expert is not complete yet and it is far beyond my expectations. Irregardless of its complexity, the final goal to be achieved, is to have HE scans form under design and generates a complete help file with automatically generated images. All what users have to do is to open generated help file and enter needed text and a few images.


In regard to ECHMRCI4.DLL file, download Help and Manual software and install, the unit should be found in system32 subdirectory
CPI now checks for "ECHMRCI4.DLL" and enable/disable related features in Help Expert  accordingly
 

List-View

CPI help expert (HE) acts as an interface between form designer and  "Help and Manual (HAM)" authoring tool.  Some screen shuts are shown on right.

HE Expected Basic Features

  1. HE expert will synchronize form under design with its help file.

  2. All help keywords and help contexts are managed by  expert. 

  3. Users do not have to worry about burden of managing help contexts numbers or help keywords.

  4. Users do not have to keep track of which component has help and which one is not. 

More functions will be applied to HE.  Current work is in its preliminary stages, so more information and details concerning this expert will follow.

 
Topics View

HE scans form under design and builds up a complete TOC of topics for all form components in a tree structure as shown in right figure.  The tree has three root nodes which are:

  1. Visual

    1. Visual node contains all topics related to form visual component. 

    2. Topics are organized based on component parent-ship hierarchy.  The node also includes topics of components used by visual components properties like popup menus, drop down menus  etc.

    3. Each node lists its component bitmap as defined by IDE.  Components that do not have bitmaps defined by IDE are supplied by bitmaps from CPI like TTabSheet, TToolButton, TComponent ...etc 

    4. HE uses bitmap for all components which do not have bitmaps defined by IDE or CPI.
       

  2. None visual
    1. None visual node contains all topics related to form none visual components
    2. Topics are organized by their components types.  For example, all topics concerning image list components are listed under "TImageList" sub-node.
  3. Support
    1. This node contains all help topics that are not related to components.

This view also uses colors for different topics as follows

  1. Red for New topics
  2. Light green for support topics
  3. Gray for selected topics
Hints-View

When you work with large forms, at the start you focus on code and avoids setting hints while programming because you feel that the hints work will distract you from concentrating on sole of your form function. As your form builds up and about to finish coding the hints problem force itself. Now, you need to go back to each component in form and set its hint through object inspector. The process of going back to each component is difficult specially if your form is a complex one like CPI form where hundreds of components scattered through a lot of tab sheet pages.

To solve this problem and reduce its effects, CPI provides a simple mechanism to help in this regard. CPI execute a search cycle on all components found in form editor and does the followings during this cycle:

  1. Checks if each component supports "hint" property
  2. If the component supports the "hint" property, CPI retrieves from component the following data
    1. component name and type and place as second column in grid as shown below.
    2. Show hint status
    3. hint content
    4. caption property if implemented by component
      It is listed to remind you of the component function in your form

After the cycle finishes, CPI organizes collected data in an editing table as shown below. In the table, CPI allows you to edit properties hint status in "show hint" column and "hint content" in hint column. CPI also allows you to save all changes made in table back to your form and to a hint status file in disk. To make your task simple, CPI provides different controls tools and flags to have full control.

Important

HE will also take a snap shut for very component with its parent and place it in help file.  This is a MAJOR step and is still under process.

Adding or removing components to form does not affect TOC.  HE will move help topics to their right place in final TOC.  Topic related to delete components are moved to "Support" node for users to review.
