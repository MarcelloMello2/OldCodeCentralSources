1-A new exploration mode
In addition to CPI Incremental (partial) and full exploration CPI added third mode of exploration called "Parentship Exploration".  In this method, CPI scan explored component and build a tree of al "TControl" components.  Figure to the right show new drop combo of explorations mode in CPI.

Figure below show exploration of BDS "AppBuilder" based on "Parentship"  exploration method.  For each node, CPI appends component name and class name.  If component name is undefined, it appends its caption if not found it appends its hint and if not found it appends 'unknown'.  Through this exploration mode CPI can rebuilt a complete duplicate copy of explored component.  It also can generate a complete table of content of a help file.  Usage of this mode is still under progress.
2-Added new locater tool  
   When exploring Delphi IDE, one need to locate found components in IDE.   This tool locates a red-rectangle that bounds component for one second.   This enables users to view component in IDE.  If component is invisible, this tool tries to make it visible and then draw red-rectangle.

Figure to right shows locating red-rectangle around "StandardToolBar" shown in exploration tree.

3-Window handle exploration. 
Child window handle exploration is used to explore window handles of any given form. Figure below shows child window handles of IDE "AppBuilder" form
4-Added "self" node to exploration tree. 

Self node now contains all properties of explored component.  CPI Explore expert now explores components into four nodes as shown in right.  These nodes are:
  1. Owned
  2. Parented
  3. Self
    Allows users to explore component internal structure in tree.  Defined class "object" properties can be explored from tree.  The internal structure is explored in four sub-nodes which are: properties, events, types, parameters
  4. Hierarchy
